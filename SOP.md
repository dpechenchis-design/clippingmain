# SOP — Як користуватися skill `clipping`

Покрокова інструкція для CP, які хочуть нарізати довге відео на короткі кліпи через Claude Code.

---

## Що цей skill робить

Бере **одне довге відео** + **транскрипт з блоками `Clip 1`, `Clip 2`...** і робить з них **готові короткі кліпи**.

Кожен блок `Clip N` у транскрипті стає **одним фінальним кліпом**, склеєним з усіх фрагментів які перераховані всередині цього блоку.

Все. Просто і чітко.

---

## Як виглядає правильний транскрипт

Скопіюй цей зразок щоб бачити структуру:

```
Clip 1 — All-or-Nothing Mentality
----------------------------------------------------------------
[00:54 → 01:06] "First fragment text..."

[03:32 → 03:35] "Second fragment text..."

[03:05 → 03:19] "Third fragment text..."

[04:09 → 04:18] "Fourth fragment text..."


Clip 2 — Track in Percentages, Not Yes/No
----------------------------------------------------------------
[04:39 → 04:50] "First fragment of clip 2..."

[05:07 → 05:13] "Second fragment of clip 2..."
```

### Правила формату

- **Заголовок кліпу** починається з `Clip ` + номер + назва через тире або двокрапку
- **Фрагмент** — це рядок з двома таймкодами у квадратних дужках: `[початок → кінець]`
- Таймкоди можна писати як `MM:SS` (наприклад `03:32`) або `HH:MM:SS` (наприклад `01:03:32`)
- Стрілка між таймкодами може бути `→`, `->`, `-`, `—`
- Цитати у лапках — необов'язкові, але корисні щоб бачити що саме у фрагменті
- Порожні рядки і лінії `------` ігноруються
- **HOOK позначка** — Claude може позначати перший фрагмент словом `HOOK` (наприклад `HOOK [00:54 → 01:06]`). Це **просто мітка для тебе**, щоб ти бачила де хук. Skill її ігнорує — для нарізки це звичайний перший фрагмент, який вже отримує буфер бо він перший.

### Що буде на виході для прикладу вище

- `Clip_1_AllOrNothingMentality.mp4` — склеєний з 4 фрагментів у порядку як у транскрипті
- `Clip_2_TrackInPercentages.mp4` — склеєний з 2 фрагментів

## Що потрібно мати на комп'ютері

Якщо ще не встановила — див. файл `Інструкція_встановлення.md`. Треба:

- **Claude Code** — для роботи з Claude локально
- **ffmpeg** — для нарізки відео
- **Skill `clipping`** — у папці `~/.claude/skills/clipping/`

Перевірити що все на місці:

```bash
claude --version
ffmpeg -version
ls ~/.claude/skills/clipping        # Mac/Linux
dir $env:USERPROFILE\.claude\skills\clipping   # Windows
```

## Як нарізати — крок за кроком

### Крок 1 — Створи робочу папку

Назвай папку зрозуміло, щоб через місяць знала що там:

**Mac:**
```bash
mkdir -p ~/Desktop/Klient_X_Episode_42
cd ~/Desktop/Klient_X_Episode_42
```

**Windows:**
```powershell
mkdir $env:USERPROFILE\Desktop\Klient_X_Episode_42
cd $env:USERPROFILE\Desktop\Klient_X_Episode_42
```

### Крок 2 — Поклади у папку файли

Через Finder/Explorer перетягни:

- Відео (наприклад `video.mov` або `interview.mp4`)
- Транскрипт (наприклад `transcript.txt` чи `transcript.md` чи `transcript.docx`)

### Крок 3 — Запусти Claude

З тієї ж папки (не з домашньої!):

```bash
claude
```

### Крок 4 — Скажи йому що робити

Скопіюй цей промпт, заміни назви файлів:

```
Наріж відео video.mov за транскриптом transcript.txt. Використай skill clipping.

Кожен блок Clip N у транскрипті це один фінальний кліп — склей всі фрагменти всередині блоку в порядку як вони йдуть у транскрипті. Буфер 5с до першого фрагменту і 1с після останнього, всередині склейки точно по таймкодах.

Збережи готові кліпи у папку clips/.
```

### Крок 5 — Подивись план

Claude покаже щось типу:

> "Я зрозумів так:
> - **Clip 1 — All-or-Nothing Mentality**: 4 фрагменти (00:54→01:06, 03:32→03:35, 03:05→03:19, 04:09→04:18). З буферами: 00:49→04:19. Загальна тривалість ~25с.
> - **Clip 2 — Track in Percentages**: 2 фрагменти...
>
> Підтверджуєш?"

**Уважно перевір:**
- Чи правильна кількість кліпів?
- Чи правильні всі фрагменти?
- Чи логічна загальна тривалість?

Якщо щось не так — кажи що саме виправити. Наприклад:
- "У Clip 1 видали третій фрагмент"
- "Додай ще один кліп з 06:00→06:30"

Тільки коли все ок — кажи `так, ріж`.

### Крок 6 — Чекай

Залежно від довжини відео і кількості кліпів — 1-5 хвилин. Claude покаже прогрес.

### Крок 7 — Перевір результат

Відкрий папку `clips/` через Finder/Explorer. Подивись кожен кліп:

- ✅ Перші слова не зрізані
- ✅ Звук синхронний з відео
- ✅ Склейки між фрагментами нормальні (не різкий обрив посеред слова)
- ✅ Назва файлу зрозуміла

Якщо щось не так — кажи Claude що саме і він перенарізає.

## Як просити перенарізати

### Якщо з'їдені перші слова

```
Перенареж Clip 2 з буфером 8 секунд до початку. Перші слова зрізані.
```

### Якщо різко обірвано в кінці

```
Перенареж Clip 1 з буфером 2 секунди після останнього фрагмента.
```

### Якщо склейки звучать дивно

```
Подивись Clip 3 — склейка між другим і третім фрагментом звучить різко. Може треба змістити таймкоди?
```

### Якщо потрібно тільки один кліп

```
Перенареж тільки Clip 2. Інші не чіпай.
```

### Якщо потрібно вищу якість

```
Перенареж усі кліпи з якістю CRF 20 (трохи вище ніж за замовчуванням 23).
```

## Часті проблеми

| Проблема | Причина | Що робити |
|----------|---------|-----------|
| Claude каже "не знаю формату транскрипту" | Транскрипт без `Clip N` заголовків | Перевір що є рядки `Clip 1`, `Clip 2`... |
| Перші слова зрізані | Буфер 5с замалий для цього відео | Кажи "збільш буфер до 8с" |
| Звук відстає | Помилка кодування | Кажи "перенареж з re-encode" |
| Помилка "input not found" | Запустила `claude` не з тієї папки | Зайди у потрібну папку через `cd` |
| `ffmpeg: command not found` | ffmpeg не встановлений | Встанови (див. інструкцію встановлення) |
| Папка `clips/` порожня після нарізки | ffmpeg помилка | Скрін у `#tech-help` чат |
| Кліп задовгий | Помилка в таймкодах транскрипту | Перевір транскрипт, скоригуй таймкоди |

---

## Чого НЕ робити

- ❌ **Не використовуй `-c copy`** для першої нарізки — зріже перші слова, попсує синхронізацію
- ❌ **Не зменшуй буфери до 0** на першому фрагменті — буде обірвано початок
- ❌ **Не клей фрагменти у хронологічному порядку** — клеїш у порядку як у транскрипті
- ❌ **Не довіряй точно таймкодам** — звіряй на слух після нарізки

## Допоміжний скрипт (опціонально)

Якщо хочеш нарізати один кліп руками без Claude, є скрипт:

```bash
bash ~/.claude/skills/clipping/scripts/cut_clip.sh video.mov 00:01:05 00:02:30 Clip_1_Test
```

Параметри: відео, початок, кінець, назва вихідного файлу.

Це для одного простого шматка — без склейки. У звичайному воркфлоу він не потрібен, Claude робить все сам.
